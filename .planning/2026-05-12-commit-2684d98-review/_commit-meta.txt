feat(plan,multi-llm): brainstorming sub-phase + file-write dispatch contract

flow:plan gains a conditional multi-LLM brainstorming sub-phase that runs
before drafting plan.md when scope warrants (always for size L; for size M
when cross-module, security-sensitive, or public-surface). Provider roles
are differentiated by lens (architecture / risk / security) so contributors
produce distinct perspectives rather than overlapping reviews.

Dogfooding the new section against itself surfaced three load-bearing
dispatch issues, fixed in references/multi-llm.md:

- File-write contract replaces stdout-capture. Contributors use their
  native Write tool to write directly to the agreed path; stdout becomes
  a diagnostic runlog only. Plays to agent CLIs' natural shape and avoids
  stdout-wrapper drift (Gemini JSON, opencode JSONL, codex prefix/suffix).
- Sentinel completion marker on the file's last line is the only reliable
  signal that a contributor finished. "Non-empty file" cannot distinguish
  "still writing" from "wrote half then died."
- Heartbeat watcher (watch_review) emits a one-line event per minute so
  hung dispatches are detectable in 60s instead of at the 600s ceiling.

Post-call verification gains two new mandatory checks (sentinel present,
structural content present) on top of the existing exit/size/signature
checks. brainstorm.md gets an idempotency precondition so resumed sessions
don't silently re-dispatch and clobber the prior synthesis.

references/models.md refreshed: gemini-3-pro-preview verified, opencode
agent-mode cost (~30k tokens per call) documented so callers can opt out.
